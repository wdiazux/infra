## Introduction

### What is Talos Linux?

Talos Linux is a modern, secure, and immutable Linux distribution designed specifically for running Kubernetes. Unlike traditional Linux distributions, Talos has no shell access, no SSH, and no package manager. All configuration and management is done declaratively through an API.

**Key Characteristics:**
- **Immutable**: Operating system cannot be modified at runtime - no package manager, no SSH shell
- **API-Driven**: All configuration and management through declarative machine configuration API
- **Kubernetes-Native**: Built from the ground up specifically for Kubernetes
- **Minimal**: Only essential components for running Kubernetes - small attack surface
- **Secure by Default**: No unnecessary services, minimal privileged components

### Why Use Talos for Kubernetes Homelab?

1. **Production-Grade Security**: Same OS used in enterprise environments, minimal attack surface
2. **GitOps-Ready**: Declarative configuration makes it perfect for Infrastructure-as-Code workflows
3. **Simplified Management**: No need to manage OS updates separately - Talos upgrades are atomic and declarative
4. **No Configuration Drift**: Immutability ensures your cluster stays exactly as configured
5. **Modern Architecture**: eBPF-based networking (Cilium), cloud-native storage (Longhorn), automated operations
6. **GPU Support**: First-class support for NVIDIA GPU passthrough for AI/ML workloads

### Prerequisites

Before starting this guide, ensure you have:

- ✅ **Talos cluster deployed** via Terraform (see Terraform documentation)

## Initial Setup Verification

### 1. Set Environment Variables

After Terraform deployment, you'll have `talosconfig` and `kubeconfig` files. Set environment variables to use them:

```bash
# Set Talos configuration
export TALOSCONFIG="/home/user/infra/terraform/talos/talosconfig"

# Set Kubernetes configuration
export KUBECONFIG="/home/user/infra/terraform/talos/kubeconfig"

# Optional: Add to ~/.bashrc or ~/.zshrc for persistence
echo 'export TALOSCONFIG="/home/user/infra/terraform/talos/talosconfig"' >> ~/.bashrc
echo 'export KUBECONFIG="/home/user/infra/terraform/talos/kubeconfig"' >> ~/.bashrc
```

**Why?**: These environment variables tell `talosctl` and `kubectl` where to find your cluster configuration files.

### 2. Verify Talos Node is Running

Check that the Talos node is accessible:

```bash
# Check Talos version and connectivity
talosctl version

# Expected output:
# Client:
#   Tag:         v1.12.1
#   SHA:         ...
#   Built:       ...
# Server:
#   NODE           TAG      SHA      BUILT
#   10.10.2.10     v1.12.1  ...      ...
```

**Success Criteria**: You should see both client and server versions displayed.

### 3. Verify Kubernetes Cluster

Check that Kubernetes is running:

```bash
# Check cluster nodes
kubectl get nodes

# Expected output:
# NAME         STATUS   ROLES           AGE   VERSION
# talos-node   Ready    control-plane   1h    v1.31.x
```

**Success Criteria**: Node should show `STATUS: Ready` and `ROLES: control-plane`.

### 4. Check Cluster Health

Verify all core components are healthy:

```bash
# Talos cluster health
talosctl health

# Check all pods in system namespaces
kubectl get pods -A

# You should see running pods in:
# - kube-system (core Kubernetes components)
# - cilium (networking)
# - longhorn-system (storage, if installed)
```

**Success Criteria**:
- `talosctl health` shows all checks passing
- All pods should be in `Running` or `Completed` state
- No pods in `CrashLoopBackOff`, `Error`, or `ImagePullBackOff`

### 5. Verify Network Configuration

Check that your network is configured correctly:

```bash
# View network interfaces on Talos node
talosctl get links

# View IP addresses
talosctl get addresses

# Expected: You should see 10.10.2.10 assigned to your network interface
```

**Success Criteria**: Talos node has IP `10.10.2.10` and can reach your network gateway.

## Essential talosctl Commands

`talosctl` is the primary tool for managing Talos nodes. Here are the essential commands:

### Cluster Information

```bash
# Check Talos and Kubernetes versions
talosctl version

# View cluster members
talosctl get members

# View cluster health (comprehensive health check)
talosctl health

# View cluster configuration
talosctl get machineconfig
```

### Monitoring and Logs

```bash
# Interactive dashboard (top-like interface)
talosctl dashboard

# View system logs (similar to journalctl)
talosctl logs

# View logs for specific service
talosctl logs controller-runtime
talosctl logs kubelet

# View kernel messages (similar to dmesg)
talosctl dmesg

# View running services
talosctl services

# View running containers (system containers)
talosctl containers

# View container logs
talosctl logs -k  # Kubernetes logs
```

### Network and System

```bash
# View network interfaces
talosctl get links

# View IP addresses
talosctl get addresses

# View routes
talosctl get routes

# Read file from Talos node
talosctl read /proc/cpuinfo
talosctl read /proc/meminfo

# List files
talosctl ls /var/log
talosctl ls /system
```

### Advanced Operations

```bash
# Restart a service
talosctl restart kubelet

# Reboot the node (graceful)
talosctl reboot

# Shutdown the node (graceful)
talosctl shutdown

# Upgrade Talos version
talosctl upgrade --image factory.talos.dev/installer/<schematic-id>:v1.12.1

# Upgrade Kubernetes version
talosctl upgrade-k8s --to 1.31.5
```

**Important**: Talos has no SSH access. All operations are done through the API via `talosctl`.

### Cluster Information

```bash
# View cluster information
kubectl cluster-info

# View cluster nodes
kubectl get nodes

# View node details
kubectl describe node talos-node

# View node resource usage (requires metrics-server)

```

### Resource Management

```bash
# List all pods in all namespaces
kubectl get pods -A

# List pods in specific namespace
kubectl get pods -n kube-system

# List all services
kubectl get svc -A

# List all deployments
kubectl get deployments -A

# List all persistent volume claims
kubectl get pvc -A

# Get detailed information about a resource
kubectl describe pod <pod-name> -n <namespace>
kubectl describe svc <service-name> -n <namespace>
```

### Working with Resources

```bash
# Apply YAML configuration
kubectl apply -f deployment.yaml

# Delete resources
kubectl delete -f deployment.yaml
kubectl delete pod <pod-name>
kubectl delete svc <service-name>

# View logs from pods
kubectl logs <pod-name>
kubectl logs <pod-name> -f  # Follow logs (like tail -f)
kubectl logs <pod-name> --previous  # Logs from previous container instance

# Execute command in pod
kubectl exec -it <pod-name> -- /bin/sh
kubectl exec <pod-name> -- ls /var/www/html

# Port forward to pod (for testing)
kubectl port-forward pod/<pod-name> 8080:80
kubectl port-forward svc/<service-name> 8080:80
```

### Troubleshooting

```bash
# View events (useful for debugging)
kubectl get events -A --sort-by='.lastTimestamp'

# Describe resources (shows detailed info and events)
kubectl describe pod <pod-name>
kubectl describe node talos-node

# Check resource requests/limits
kubectl get pods -o=jsonpath='{range .items[*]}{.metadata.name}{"\t"}{.spec.containers[*].resources}{"\n"}{end}'

# Check which node a pod is running on
kubectl get pods -o wide
```

## First Service Deployment - NGINX Example

Let's deploy a simple nginx web server to verify your cluster is working correctly. This example demonstrates:
- Creating a deployment
- Exposing services internally (ClusterIP)
- Exposing services externally (LoadBalancer with Cilium L2)
- Testing and cleanup

### Step 1: Create NGINX Deployment YAML

Create a file named `nginx-demo.yaml`:

```yaml
# nginx-demo.yaml
# Complete NGINX deployment with ClusterIP and LoadBalancer services
---
# Deployment: Defines the nginx pods
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-demo
  namespace: default
  labels:
    app: nginx
    environment: demo
spec:
  replicas: 2  # Run 2 pods for basic redundancy
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
        environment: demo
    spec:
      containers:
      - name: nginx
        image: nginx:1.27-alpine  # Lightweight Alpine-based image
        ports:
        - containerPort: 80
          name: http
        resources:
          requests:
            memory: "64Mi"   # Minimum memory needed
            cpu: "100m"      # 0.1 CPU cores minimum
          limits:
            memory: "128Mi"  # Maximum memory allowed
            cpu: "200m"      # 0.2 CPU cores maximum
        livenessProbe:
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 10
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /
            port: 80
          initialDelaySeconds: 5
          periodSeconds: 5
---
# Service (ClusterIP): Internal-only access
# This service is only accessible from within the cluster
apiVersion: v1
kind: Service
metadata:
  name: nginx-clusterip
  namespace: default
  labels:
    app: nginx
spec:
  type: ClusterIP
  selector:
    app: nginx
  ports:
    - port: 80
      targetPort: 80
      protocol: TCP
      name: http
---
# Service (LoadBalancer): External access via Cilium L2
# Cilium will assign an IP from the pool (10.10.2.240-10.10.2.254)
apiVersion: v1
kind: Service
metadata:
  name: nginx-loadbalancer
  namespace: default
  labels:
    app: nginx
spec:
  type: LoadBalancer
  selector:
    app: nginx
  ports:
    - port: 80
      targetPort: 80
      protocol: TCP
      name: http
  # Optional: Request a specific IP from the pool
  # loadBalancerIP: 10.10.2.240
```

**What This Creates**:
- **Deployment**: 2 nginx pods with health checks and resource limits
- **ClusterIP Service**: Internal service accessible only within the cluster
- **LoadBalancer Service**: External service accessible from your network via Cilium L2

### Step 2: Deploy to Cluster

Apply the configuration:

```bash
# Deploy nginx
kubectl apply -f nginx-demo.yaml

# Expected output:
# deployment.apps/nginx-demo created
# service/nginx-clusterip created
# service/nginx-loadbalancer created
```

### Step 3: Verify Deployment

Check that pods are running:

```bash
# View pods
kubectl get pods -l app=nginx

# Expected output:
# NAME                          READY   STATUS    RESTARTS   AGE
# nginx-demo-xxxxxxxxxx-xxxxx   1/1     Running   0          30s
# nginx-demo-xxxxxxxxxx-xxxxx   1/1     Running   0          30s

# View detailed pod information
kubectl get pods -l app=nginx -o wide

# This shows which node each pod is on (both on talos-node in single-node setup)
```

**Success Criteria**: Both pods should show `STATUS: Running` and `READY: 1/1`.

### Step 4: Verify Services

Check that services are created:

```bash
# View all services
kubectl get svc

# Expected output:
# NAME                  TYPE           CLUSTER-IP      EXTERNAL-IP    PORT(S)        AGE
# kubernetes            ClusterIP      10.96.0.1       <none>         443/TCP        1h
# nginx-clusterip       ClusterIP      10.96.xxx.xxx   <none>         80/TCP         1m
# nginx-loadbalancer    LoadBalancer   10.96.xxx.xxx   10.10.2.240    80:30080/TCP   1m
```

**Success Criteria**:
- `nginx-clusterip` has CLUSTER-IP but no EXTERNAL-IP (internal only)
- `nginx-loadbalancer` has EXTERNAL-IP assigned from Cilium L2 pool (10.10.2.240-254)

**Note**: If `nginx-loadbalancer` shows `<pending>` for EXTERNAL-IP, see the [Troubleshooting](#troubleshooting) section.

### Step 5: Test Internal Access (ClusterIP)

Test the ClusterIP service from within the cluster:

```bash
# Port forward to test locally
kubectl port-forward svc/nginx-clusterip 8080:80

# In another terminal, test with curl
curl http://localhost:8080

# Expected output: nginx welcome page HTML

# Press Ctrl+C to stop port forwarding
```

**Success Criteria**: You should see the nginx HTML welcome page.

### Step 6: Test External Access (LoadBalancer)

Test the LoadBalancer service from your network:

```bash
# Get the LoadBalancer IP
LB_IP=$(kubectl get svc nginx-loadbalancer -o jsonpath='{.status.loadBalancer.ingress[0].ip}')
echo "LoadBalancer IP: $LB_IP"

# Test from command line
curl http://$LB_IP

# Expected output: nginx welcome page HTML
```

**From Your Browser**:
1. Open browser
2. Navigate to `http://10.10.2.240` (or whatever IP was assigned)
3. You should see the nginx welcome page

**Success Criteria**:
- You can curl the LoadBalancer IP and get nginx HTML
- You can access the IP from your browser on the same network

### Step 7: Verify Load Balancing

The service has 2 pods. Let's verify load balancing works:

```bash
# Get pod names
kubectl get pods -l app=nginx

# View logs from both pods
kubectl logs -l app=nginx --tail=10

# Make multiple requests
for i in {1..10}; do
  curl -s http://10.10.2.240 > /dev/null
  echo "Request $i sent"
  sleep 1
done

# Check logs again - both pods should show access logs
kubectl logs -l app=nginx --tail=20
```

**Success Criteria**: Both pods should show access log entries, indicating load balancing is working.

### Step 8: Scale the Deployment

Let's scale up the deployment:

```bash
# Scale to 4 replicas
kubectl scale deployment nginx-demo --replicas=4

# Watch pods being created
kubectl get pods -l app=nginx -w

# Press Ctrl+C when all 4 pods are Running

# Verify scaling
kubectl get pods -l app=nginx
```

**Success Criteria**: You should see 4 nginx pods running.

### Step 9: Cleanup

When done testing, remove all resources:

```bash
# Delete all resources created by the YAML file
kubectl delete -f nginx-demo.yaml

# Expected output:
# deployment.apps "nginx-demo" deleted
# service "nginx-clusterip" deleted
# service "nginx-loadbalancer" deleted

# Verify cleanup
kubectl get pods -l app=nginx
kubectl get svc -l app=nginx

# Should show: No resources found
```

**Success Criteria**: All nginx resources are removed cleanly.

## Troubleshooting

### Issue: Pods Stuck in "Pending" State

**Symptoms**:
```bash
kubectl get pods
# NAME                          READY   STATUS    RESTARTS   AGE
# nginx-demo-xxxxxxxxxx-xxxxx   0/1     Pending   0          5m
```

**Diagnosis**:
```bash
# Check pod events
kubectl describe pod <pod-name>

# Common causes:
# - Insufficient resources (CPU/memory)
# - No storage available (if using PVC)
# - Node not ready
```

**Solution**:
```bash
# Check node resources
kubectl describe node talos-node

# Check if node has enough CPU/memory
kubectl top node talos-node

# For single-node cluster, you may need to remove control-plane taint:
kubectl taint nodes --all node-role.kubernetes.io/control-plane-
```

### Issue: Pods in "ImagePullBackOff" State

**Symptoms**:
```bash
kubectl get pods
# NAME                          READY   STATUS             RESTARTS   AGE
# nginx-demo-xxxxxxxxxx-xxxxx   0/1     ImagePullBackOff   0          5m
```

**Diagnosis**:
```bash
# Check pod events
kubectl describe pod <pod-name>

# Look for errors like:
# - Failed to pull image "nginx:1.27-alpine"
# - ErrImagePull
```

**Solution**:
```bash
# Check Talos node can reach internet
talosctl get links
talosctl get routes

# Test DNS resolution
kubectl run -it --rm debug --image=busybox --restart=Never -- nslookup google.com

# Check container runtime
talosctl containers -k
```

### Issue: Pods in "CrashLoopBackOff" State

**Symptoms**:
```bash
kubectl get pods
# NAME                          READY   STATUS             RESTARTS   AGE
# nginx-demo-xxxxxxxxxx-xxxxx   0/1     CrashLoopBackOff   5          5m
```

**Diagnosis**:
```bash
# Check pod logs
kubectl logs <pod-name>

# Check previous container logs (if restarted)
kubectl logs <pod-name> --previous

# Describe pod for events
kubectl describe pod <pod-name>
```

**Solution**:
- Check application configuration
- Verify image is correct
- Check resource limits (may be too low)
- Review application startup logs

### Issue: LoadBalancer Service Has No EXTERNAL-IP

**Symptoms**:
```bash
kubectl get svc nginx-loadbalancer
# NAME                 TYPE           CLUSTER-IP      EXTERNAL-IP   PORT(S)        AGE
# nginx-loadbalancer   LoadBalancer   10.96.xxx.xxx   <pending>     80:30080/TCP   5m
```

**Diagnosis**:
```bash
# Check Cilium status
kubectl get pods -n cilium

# Check Cilium L2 announcement policy
kubectl get l2advertisementpolicy -A
kubectl get ciliumloadbalancerippools -A

# Verify Cilium L2 configuration
kubectl get configmap cilium-config -n cilium -o yaml | grep -A 5 l2
```

**Solution**:
```bash
# Verify L2 announcement policy exists
kubectl get l2advertisementpolicy -A

# If missing, check Terraform deployment:
# terraform/kubernetes/cilium-l2-config.tf should create:
# - CiliumLoadBalancerIPPool (10.10.2.240/28)
# - L2AnnouncementPolicy

# Re-apply Cilium L2 configuration
cd /home/user/infra/terraform/kubernetes
terraform apply -target=kubectl_manifest.cilium_ippool
terraform apply -target=kubectl_manifest.cilium_l2_policy

# Restart Cilium pods if needed
kubectl rollout restart daemonset/cilium -n cilium
```

### Issue: Cannot Access LoadBalancer IP from Network

**Symptoms**:
- `kubectl get svc` shows EXTERNAL-IP assigned
- `curl http://10.10.2.240` times out or connection refused

**Diagnosis**:
```bash
# Check if service has endpoints
kubectl get endpoints nginx-loadbalancer

# Should show pod IPs, e.g.:
# NAME                 ENDPOINTS                     AGE
# nginx-loadbalancer   10.244.0.5:80,10.244.0.6:80   5m

# Test from Talos node
talosctl -n 10.10.2.10 read /proc/net/arp

# Check Cilium L2 announcements
kubectl logs -n cilium -l k8s-app=cilium | grep -i l2
```

**Solution**:
```bash
# Verify network connectivity to Talos node
ping 10.10.2.10

# Verify LoadBalancer IP is in correct subnet
# Should be in 10.10.2.240-254 range

# Check if IP is announced via ARP
# From another machine on network:
arp -a | grep 10.10.2.240

# Verify Cilium is announcing the IP
kubectl exec -n cilium -it <cilium-pod> -- cilium service list

# Check firewall on Talos node (should be minimal)
talosctl get links
```

### Issue: Cannot View Logs or Events

**Symptoms**:
```bash
kubectl logs <pod-name>
# Error from server (BadRequest): container "nginx" in pod "nginx-demo-xxx" is waiting to start
```

**Solution**:
```bash
# Wait for pod to be Running
kubectl get pods -w

# For pods that never start, check events:
kubectl describe pod <pod-name>

# Check node logs
talosctl logs -f
```
